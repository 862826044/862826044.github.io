Cargo geiger分析工具

2021-04-08

USAGE:

cargo geiger [OPTIONS]

OPTIONS:

-p, –package Package to be used as the root of the tree.包将用作树的根。

​ –features Space-separated list of features to activate.以空格分隔的要激活的功能列表

​ –all-features Activate all available features.激活所有可用功能

​ –no-default-features Do not activate the default feature.不激活“默认”功能

​ –target Set the target triple.

​ –all-targets Return dependencies for all targets. By default only the host target is matched.返回所有目标的依赖关系。 默认情况下，仅匹配主机目标

​ –manifest-path Path to Cargo.toml.

-i, –invert Invert the tree direction.

​ –no-indent Display the dependencies as a list (rather than a tree).

​ –prefix-depth Display the dependencies as a list (rather than a tree), but prefixed with the depth.以深度为前缀

-a, –all Don’t truncate dependencies that have already been displayed.不要截断已经显示的依赖项。

​ –charset Character set to use in output: utf8, ascii[default: utf8].

​ –format Format string used for printing dependencies[default: {p}].

-v, –verbose Use verbose output (-vv very verbose/build.rs output).使用详细输出

-q, –quiet No output printed to stdout other than the tree.除了树以外，没有输出输出到stdout

​ –color Coloring: auto, always, never.

​ –frozen Require Cargo.lock and cache are up to date.要求Cargo.lock和缓存是最新的

​ –locked Require Cargo.lock is up to date.

​ –offline Run without accessing the network.

-Z "…" Unstable (nightly-only) flags to Cargo.

​ –include-tests Count unsafe usage in tests..

​ –build-dependencies Also analyze build dependencies.还分析构建依赖关系

​ –dev-dependencies Also analyze dev dependencies.

​ –all-dependencies Analyze all dependencies, including build and dev.

​ –forbid-only Don’t build or clean anything, only scan entry point .rs source files for. forbid(unsafe_code) flags. This is significantly faster than the default scanning mode. TODO: Add ability to combine this with a whitelist for use in CI.不构建或清除任何内容，仅扫描入口点.rs源文件。 forbid（unsafe_code）标志。 这比默认扫描模式快得多。 待办事项：添加将其与白名单结合在一起以在CI中使用的功能。

-h, –help Prints help information.

-V, –version Prints version information.
